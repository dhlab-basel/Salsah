Developer Guidelines¶

Dear Developer, this guideline should help you and us to have always a similar code and file structure. Both definition are defined on the following pages.

Code structure¶

We’re all developing salsah with Angular_CLI and the WebStorm editor by JetBrains. This editor (but also many others) is able to read the .editorconfig file in the Salsah root directory. It’s a configuration definition for the typescript- and the scss-lint tools. The following recommendations are important:

General we’re using spaces instead of tabs (indent_style) with a size of 4 (indent_size). The styling files, where we’re using scss (sass), have an exception of the indent size, namely 2.

Typescript Code Styling We’re following the TSLint recommendation. Please set your editor configuration as described there. It helps on reformat code (one cool function in WebStorm). Some important points are:
